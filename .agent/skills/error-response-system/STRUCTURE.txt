FILE PURPOSES
=============

ROOT LEVEL
----------
README.txt
  Purpose: Package overview, installation instructions
  Audience: Everyone (first file to read)
  
SKILL.md
  Purpose: Main skill documentation with auto-activation rules
  Audience: AI Agents & Developers
  Key Features: Decision trees, pattern recognition, mandatory rules


ASSETS FOLDER
-------------
error-codes-template.json
  Purpose: Complete error codes database with metadata
  Contains: HTTP errors, Business errors (AUTH, ORD, PAY, etc.)
  Metadata: useCases, relatedCodes, _aiGuidance
  

SCRIPTS FOLDER
--------------
validate-error-codes.py
  Purpose: Validate error codes JSON structure
  Features: Severity levels, detailed error messages, Vietnamese check
  Usage: python scripts/validate-error-codes.py
  
generate-types.py
  Purpose: Generate TypeScript types from JSON
  Output: types.ts with interfaces, unions, helpers
  Usage: python scripts/generate-types.py


REFERENCES FOLDER
-----------------
README.md
  Purpose: Quick start guide and overview
  Audience: All users
  Contains: Installation, examples, best practices
  
AI-AGENT-GUIDE.md
  Purpose: Complete instructions for AI agents
  Audience: Claude, GPT-4, other AI assistants
  Contains: Auto-activation logic, decision trees, patterns
  
implementation-guide.md
  Purpose: Detailed implementation instructions
  Audience: Developers
  Contains: Setup, helper functions, frontend integration, testing
  
migration-guide.md
  Purpose: Upgrade guide from v1.x to v2.0
  Audience: Existing users
  Contains: Migration steps, breaking changes (none!), enhancements


WORKFLOW
========

FOR AI AGENTS
-------------
1. Read SKILL.md for auto-activation rules
2. Consult AI-AGENT-GUIDE.md for decision logic
3. Use pattern recognition to select error codes
4. Generate type-safe error responses

FOR DEVELOPERS
--------------
1. Read README.txt for overview
2. Follow implementation-guide.md for setup
3. Copy error-codes-template.json to project
4. Run generate-types.py
5. Create helper functions
6. Start using in API routes

FOR MIGRATION
-------------
1. Read migration-guide.md
2. Update error codes JSON (optional)
3. Replace scripts
4. Regenerate types
5. Done! (no code changes needed)
