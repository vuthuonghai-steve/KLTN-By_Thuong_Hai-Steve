# Interaction Protocol

## Ask The User When
- Input is insufficient to proceed.
- Multiple interpretations exist.
- The action is risky or irreversible.
- Confidence is below 70%.

## Phase-Based Behavior
- During Research and Design, you MUST ask clarifying questions when ambiguity exists.
- During Build and Verify, do not ask new questions unless blocked; proceed and confirm after completion.

## Scope or Architecture Changes
- If a decision would change architecture or scope, pause and ask for explicit approval.
