# Implementation Checklist

## Phase: Research
- [ ] Input is clearly identified.
- [ ] Tools to use are listed.
- [ ] Potential blind spots are identified.

## Phase: Design
- [ ] Workflow or flowchart is defined.
- [ ] Interaction points are defined.
- [ ] Output format is defined.

## Phase: Build
- [ ] Directory structure uses valid zones only.
- [ ] `SKILL.md` is created or updated.
- [ ] Required resources are added to zones.

## Phase: Verify
- [ ] Test cases are run.
- [ ] Checklist is verified.
- [ ] Failures are fixed or rolled back.

## Phase: Maintenance
- [ ] Feedback is captured.
- [ ] Version control is updated.
- [ ] Documentation is updated.
